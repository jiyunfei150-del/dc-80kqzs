流程详细说明
🗂
一、分级脱敏阶段
  • 建立映射表，包含字段：database_name、table_name、level、luanshen_table_name、union_id
  • 数据级别划分：
    级别说明策略
    1级最高级禁止数据外泄
    2级次高级脱敏后可传出
    3级普通级正常传出
👤
二、用户请求阶段
  • 用户使用 OpenClaw 提问时，请求经过财务后台 API
  • 开发根据请求 IP 判断是 常规访问 还是 AI访问（当前AI为OpenClaw）
  • AI访问走独立服务链路，分配数据库 只读账号
🛡
三、安全检测 · 第一道关卡
  • AI发起查询时，检测 SQL 中涉及的表级别：
    — 级别3 或 级别2且已脱敏 → 放行
    — 级别1 或 级别2且未脱敏 → 打回
  • 放行后，将 SQL 中需要脱敏的表替换为 孪生视图，修改 SQL 并执行，获取脱敏后的数据文件
🔍
四、安全检测 · 第二道关卡
  • 对数据文件进行二次扫描，检查是否包含以下敏感信息：
  • 身份证号  手机号  公司主体  等极敏感数据
  • 若发现敏感数据 → 打回，阻止数据交付
📊
五、数据交付阶段
  • 通过全部安全检测后，用户将获得 脱敏后数据 的AI分析结果
  • 数据全程未暴露原始敏感信息
🔓
六、解密平台（按需）
  • 提供独立的解密平台，用户可将脱敏分析结果传入进行 反脱敏
  • 此功能按需开放，需额外权限控制
