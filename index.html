流程详细说明
1
AI-1 · 查询重写器
  • 接收用户原始提问，加载 entity_mapping 作为上下文
  • 识别实体名（包括缩写、别名：跃动 = 跃动小子）
  • 统一业务术语：平台 / 信道 → 渠道
  • 输出脱敏后标准化提问，传给 AI-2
  • AI-1 只做翻译，不接触任何业务数据
2
AI-2 · 数据分析器
  • 接收 AI-1 输出的脱敏提问，永远看不到真实实体名
  • 生成 SQL → 查 schema_registry 判断表级别
  • L2 表替换为孪生视图，StarRocks 原生加密函数实现
  • SQL 表替换由小伙伴实现，AI-2 直接使用
  • 分析完成后经二次扫描，通过则返回结果
3
数据分级与孪生视图
  • 建立映射表：database_name、table_name、level、孪生视图名
  • L1：最高级，禁止数据外泄，直接拦截
  • L2：次高级，替换孪生视图，加密值转确定性标签
  • L3：普通级，金额/数量等指标正常传出
4
二次扫描 · 安全保底
  • 对 AI-2 输出的数据文件进行扫描
  • 检测：身份证号、手机号、公司主体、真实名称
  • 若发现敏感数据 → 打回，阻止数据交付
  • 通过全部检测后，用户获得分析结果
